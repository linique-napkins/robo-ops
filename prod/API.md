# SO101 Demo Server API

Base URL: `http://<host>:8000`

Interactive docs (auto-generated by FastAPI): `http://<host>:8000/docs`

## State Machine

```
DISCONNECTED ──connect──▶ IDLE ──▶ { REPLAYING | INFERRING | TELEOP }
                           ▲                    │
                           └──── STOWING ◀──────┘
```

| State | Description |
|-------|-------------|
| `disconnected` | No robot connection. Only `/api/connect` and `/api/state` respond. |
| `idle` | Robot connected, cameras streaming, no actions sent. |
| `replaying` | Playing back a demo recording from `demo/recordings/`. |
| `inferring` | Running a trained policy. Action = model prediction each frame. |
| `teleop` | Receiving leader positions over `/ws/teleop`, forwarding to follower. |
| `stowing` | Moving arms to safe position (~1.7 s). Transitions to `idle`. |

Any active state can be stopped (returns to `idle`) or paused (cameras keep streaming, actions freeze).

---

## REST Endpoints

All mutating endpoints return:

```json
{ "ok": true, "state": { ... } }
```

On error:

```json
{ "ok": false, "error": "description" }
```

with an appropriate HTTP status code (404 for not-found, 500 for internal errors).

### GET /api/state

Current server state plus available models, recordings, and cameras.

**Response:**

```json
{
  "state": "idle",
  "paused": false,
  "operation": null,
  "available_models": {
    "act-fold": "ACT Full Fold",
    "pi0-pickup": "Pi0 Pickup"
  },
  "available_recordings": [
    "best",
    "demo",
    "linique_unfold_3"
  ],
  "available_cameras": ["top", "left", "right"]
}
```

| Field | Type | Description |
|-------|------|-------------|
| `state` | string | Current state machine value. |
| `paused` | bool | Whether the active operation is paused. |
| `operation` | string\|null | Name of the running operation (`"replay"`, `"inference"`, `"teleop"`), or `null`. |
| `available_models` | object | Model key → display name, from `prod/config.toml`. |
| `available_recordings` | string[] | JSON filenames (without `.json`) in `demo/recordings/`. |
| `available_cameras` | string[] | Camera names configured in `config.toml` (empty when disconnected). |

---

### POST /api/connect

Connect to robot hardware (follower arms + cameras) and start the 30 fps observation loop. Cameras begin streaming immediately.

**Requires state:** `disconnected`
**Transitions to:** `idle`

---

### POST /api/disconnect

Stop any active operation, stow arms to safe position, then disconnect hardware.

**Requires state:** any connected state
**Transitions to:** `disconnected`

---

### POST /api/stow

Move both arms to the safe stow position (50 interpolation steps at 30 fps, ~1.7 s). Stops any running operation first.

**Requires state:** any connected state
**Transitions to:** `stowing` → `idle`

---

### POST /api/pause

Toggle pause on the active operation. When paused, the robot holds its current position and cameras keep streaming.

**Requires state:** `replaying`, `inferring`, or `teleop`

**Response** (additional field):

```json
{ "ok": true, "paused": true, "state": { ... } }
```

---

### POST /api/replay/{name}

Start replaying a demo recording.

**Path parameters:**

| Param | Type | Description |
|-------|------|-------------|
| `name` | string | Recording filename without `.json` (e.g. `linique_unfold_3`). |

**Query parameters:**

| Param | Type | Default | Description |
|-------|------|---------|-------------|
| `loop` | bool | `false` | Loop playback when the recording ends. |

**Requires state:** `idle`
**Transitions to:** `replaying`

**Example:**

```
POST /api/replay/linique_unfold_3?loop=true
```

---

### POST /api/replay/{name}/stop

Stop the current replay and return to idle. The `{name}` segment is required by the URL pattern but is not used; the currently running replay is stopped regardless.

**Requires state:** `replaying`
**Transitions to:** `idle`

---

### POST /api/infer/{model}

Load a trained policy and start inference. The model is loaded synchronously on this request (may take several seconds for large models), then the control loop runs predictions at 30 fps.

**Path parameters:**

| Param | Type | Description |
|-------|------|-------------|
| `model` | string | Model key from `prod/config.toml` (e.g. `act-fold`). |

**Requires state:** `idle`
**Transitions to:** `inferring`

**Example:**

```
POST /api/infer/act-fold
```

---

### POST /api/infer/{model}/stop

Stop inference and return to idle.

**Requires state:** `inferring`
**Transitions to:** `idle`

---

### POST /api/teleop/start

Enter remote teleoperation mode. The server waits for a client to connect to `/ws/teleop` and send leader arm positions.

**Requires state:** `idle`
**Transitions to:** `teleop`

---

### POST /api/teleop/stop

Exit teleop mode and return to idle.

**Requires state:** `teleop`
**Transitions to:** `idle`

---

## Camera Streams

### GET /stream/{camera}

MJPEG stream over `multipart/x-mixed-replace`. Works natively in all browsers including iPad Safari.

**Path parameters:**

| Param | Type | Description |
|-------|------|-------------|
| `camera` | string | Camera name: `left`, `right`, or `top`. |

**Usage in HTML:**

```html
<img src="/stream/left">
<img src="/stream/top">
<img src="/stream/right">
```

Streams at ~30 fps, ~30-50 KB per frame (640x480 JPEG, quality 80). Cameras stream in all connected states including `idle`.

## WebSocket Endpoints

### WS /ws/state

Real-time state broadcast. Receives a JSON state object on connect and on every state change.

**Message format** (server → client):

```json
{
  "state": "replaying",
  "paused": false,
  "operation": "replay"
}
```

The client does not need to send messages; the connection is kept alive by the WebSocket protocol. Any text sent by the client is ignored.

**Example (JavaScript):**

```javascript
const ws = new WebSocket("ws://localhost:8000/ws/state");
ws.onmessage = (e) => {
  const state = JSON.parse(e.data);
  console.log(state.state);  // "idle", "replaying", etc.
};
```

---

### WS /ws/teleop

Remote leader arm control. The client sends joint positions at 30 Hz; the server forwards them to the follower arms.

**Requires:** `POST /api/teleop/start` must be called first to enter teleop mode.

**Message format** (client → server):

```json
{
  "positions": {
    "left_shoulder_pan.pos": 0.0,
    "left_shoulder_lift.pos": -45.2,
    "left_elbow_flex.pos": 90.0,
    "left_wrist_flex.pos": -30.0,
    "left_wrist_roll.pos": 0.0,
    "left_gripper.pos": 50.0,
    "right_shoulder_pan.pos": 0.0,
    "right_shoulder_lift.pos": -45.2,
    "right_elbow_flex.pos": 90.0,
    "right_wrist_flex.pos": -30.0,
    "right_wrist_roll.pos": 0.0,
    "right_gripper.pos": 50.0
  }
}
```

All 12 joint positions (6 per arm) in degrees. If no message arrives for 2 seconds, the follower holds its last position.

**Standalone client:**

```bash
uv run prod/teleop_client.py --server 100.x.y.z:8000
```

This reads physical leader arms via USB serial and streams positions over the WebSocket.

## Model Configuration

Models are registered in `prod/config.toml`:

```toml
[models.act-fold]
display_name = "ACT Full Fold"
policy_repo_id = "data/outputs/jhimmens_linique-act-v2/pretty-elevator-4/final"
dataset_repo_id = "jhimmens/linique-v2"
task = "full-fold-v1"
device = "cuda"
temporal_ensemble_coeff = 0.01

[models.pi0-pickup]
display_name = "Pi0 Pickup"
policy_repo_id = "data/outputs/pi0-pickup/final"
dataset_repo_id = "jhimmens/linique-v2-pickup"
task = "pickup"
device = "cuda"
# backend = "remote"
# server_url = "https://xxx.runpod.ai"
```

| Field | Required | Description |
|-------|----------|-------------|
| `display_name` | no | Human-readable name shown in the UI. Defaults to the key. |
| `policy_repo_id` | yes | Local path or HuggingFace repo ID for the trained policy. |
| `dataset_repo_id` | yes | Dataset used for training (provides feature definitions). |
| `task` | yes | Task name (must match training config). |
| `device` | no | `"cuda"`, `"mps"`, or `"cpu"`. Default: `"cuda"`. |
| `temporal_ensemble_coeff` | no | Smoothing coefficient for overlapping action chunks. |
| `backend` | no | `"local"` (default) or `"remote"` for cloud inference. |
| `server_url` | if remote | URL of the remote inference server. |

## Typical Workflow

```bash
# 1. Start the server
uv run prod/server.py

# 2. Open browser
open http://localhost:8000

# 3. Connect robot (or click Connect in UI)
curl -X POST http://localhost:8000/api/connect

# 4. Camera feeds appear automatically in <img> tags

# 5. Play a demo recording
curl -X POST http://localhost:8000/api/replay/linique_unfold_3?loop=true

# 6. Stop and switch to inference
curl -X POST http://localhost:8000/api/replay/linique_unfold_3/stop
curl -X POST http://localhost:8000/api/infer/act-fold

# 7. Pause/resume
curl -X POST http://localhost:8000/api/pause

# 8. Stow and disconnect
curl -X POST http://localhost:8000/api/disconnect
```
